ULTIMA 2.0
Text-to-Image Diagram Prompt Pack
Phase 1 - Scheduler, Semaphore, and Memory Manager

Purpose

Use these prompts to generate a matched set of technical diagrams for the ULTIMA 2.0 Phase 1 repository. The diagrams should look like engineering infographics, not fantasy art, concept art, or generic app mockups.

Global Style Instructions For Every Diagram

- Create a clean technical infographic with crisp vector lines, sharp labels, and strong hierarchy.
- Use a 16:9 landscape composition at high resolution.
- Prefer a white or very light background with dark navy linework, charcoal text, and restrained accent colors such as teal, orange, and muted red.
- Use short, readable labels in large sans-serif text blocks. Avoid dense paragraphs inside the image.
- Favor boxes, swimlanes, arrows, state nodes, and callout panels over decorative visuals.
- Make the diagram look like a polished software architecture poster for an operating-systems course project.
- Include a title block at the top and a small legend block at the bottom right.
- Keep the visual language consistent across the entire set so the diagrams look like one family.
- Do not add unrelated UI chrome, laptops, people, mascots, photorealistic textures, or abstract art backgrounds.
- Do not invent extra subsystems beyond what is named in the prompt.

Negative Prompt For Every Diagram

- no photorealism
- no 3D rendering
- no fantasy art
- no glowing cyberpunk city scenes
- no unnecessary characters
- no stock-photo office scenes
- no clutter
- no low-contrast text
- no handwritten fonts
- no gibberish labels
- no random code snippets

Diagram 3
Memory Manager Operation Diagram

Prompt:

Create a technical explainer infographic titled "Mem_mgr First-Fit Heap Behavior". Use a left-to-right sequence showing four stages of the heap. Stage 1: one free segment from 0 to 1023. Stage 2: Task_A allocates 160 bytes at offset 0, Task_B allocates 112 bytes at offset 160, Task_C allocates 96 bytes at offset 272, remaining space free from 368 to 1023. Stage 3: Task_A frees its buffer, leaving a free segment at 0 to 159 while Task_B and Task_C remain allocated. Stage 4: Task_B and Task_C free their buffers and the heap coalesces back to one free segment from 0 to 1023. Include a small side panel listing the methods reset, allocate, free_block, write, read, dump. Add callouts for deterministic first-fit scan, zeroing on free, stable offset addresses, and free-segment coalescing. Use color coding: free segments in light gray, Task_A in teal, Task_B in orange, Task_C in muted red. Make the whole diagram read like an operating-systems memory-management poster.

Diagram 4
Semaphore FIFO Behavior Diagram

Prompt:

Create an engineering flow diagram titled "Semaphore FIFO Queue and Wake-Up Order". Show a binary semaphore protecting Printer_Output. Use a time-ordered sequence from top to bottom. First show Task_A calling down and acquiring the resource. Then show Task_B calling down and moving into BLOCKED state, entering the wait queue first. Then show Task_C calling down and entering the wait queue behind Task_B. Next show Task_A calling up, the semaphore waking Task_B, ownership transferring to Task_B, and Task_C remaining blocked. Finally show Task_B calling up and waking Task_C. Include a side box listing semaphore fields: resource_name, value, owner_task_id, wait_queue, down_operations, up_operations, contention_events, last_transition. Make the queue visually obvious with FIFO arrows and emphasize that there is no busy waiting. Use short labels like down, blocked, queued, wake, owner transfer, and available.

Diagram 5
Scheduler Round-Robin And State Transition Diagram

Prompt:

Create a state-and-flow infographic titled "Scheduler Round-Robin Execution and Task States". Show the Scheduler scanning READY tasks in round-robin order and moving them through READY, RUNNING, BLOCKED, READY, and DEAD. Use three task lanes labeled Task_A, Task_B, and Task_C. Show Task_A moving READY to RUNNING to READY to RUNNING to DEAD. Show Task_B and Task_C moving READY to RUNNING to BLOCKED, then waking back to READY, then RUNNING, then DEAD. Add a side panel for the Scheduler class showing process_table, current_running_task, next_task_id, scheduler_tick, last_scheduler_event, and the key methods create_task, yield, block_task, unblock_task, kill_task, and garbage_collect. Include a small process-table callout and a run-queue summary strip. The look should be precise, deterministic, and focused on scheduling logic rather than generic workflow art.

Recommended Workflow

1. Generate all six diagrams with the same global style instructions.
2. Keep color assignments stable across the set.
3. Reject any output that contains unreadable or hallucinated labels.
4. If the generator struggles with text, regenerate with fewer words inside the image and keep the same structure.
5. After generation, add final labels manually in an image editor if necessary for submission quality.
