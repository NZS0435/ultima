ULTIMA 2.0
Phase 1 Abstract
Scheduler, Semaphore, and Memory Manager

Phase 1 of ULTIMA 2.0 implements the execution-control layer required by the assignment: a dynamic scheduler, a binary semaphore, and a deterministic memory manager working together in one demonstrable system. The scheduler owns a process table of Task Control Blocks and performs strict round-robin dispatch across READY tasks. Each task moves through the required state model of RUNNING, READY, BLOCKED, and DEAD, and completed tasks are removed through garbage collection. The semaphore protects a single shared resource named Printer_Output and uses a FIFO wait queue so contention is resolved by blocking and waking tasks rather than busy waiting. The memory manager uses a deterministic first-fit heap layout so each task can reserve and release an explicit buffer during the proof run.

The demonstration scenario uses three tasks and three task-owned buffers. Task_A reserves a work buffer, acquires the shared resource first, and yields while still holding it. Task_B reserves a wait-record buffer, requests the same resource, and becomes BLOCKED. Task_C reserves its own wait-record buffer, requests the resource after Task_B, and becomes BLOCKED behind it in the queue. When Task_A releases the resource and frees its buffer, Task_B is awakened first, then Task_C, proving both correct blocking behavior and FIFO wake-up order while the heap transitions from full use back to a single free segment.

To support grading and presentation, the phase includes a multi-window terminal interface that keeps the scheduler panel, semaphore panel, state/race/memory proof panel, per-task panels, log window, and controls visible together. After the final task states are shown, the program can export a transcript that records the same execution as plain text, including scheduler, semaphore, and memory-manager dumps. This phase therefore demonstrates the assignment requirements in both live visual form and written evidence form while preserving a cross-platform build path for macOS, Windows, and Cygwin collaborators.
