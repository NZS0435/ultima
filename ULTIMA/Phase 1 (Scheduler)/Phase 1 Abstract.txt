ULTIMA 2.0: Phase 1 Abstract

This initial phase of the ULTIMA 2.0 operating system project establishes the essential process management and resource synchronization subsystems necessary for cooperative (non-preemptive) multitasking. The primary technical objectives were the implementation of a centralized round-robin scheduler and a robust binary semaphore mechanism to handle shared I/O resources.

The scheduler governs thread execution via a dynamically allocated Task Control Block (TCB) architecture, maintaining precise control over four distinct process states: RUNNING, READY, BLOCKED, and DEAD. By executing controlled context switching, the scheduler fairly delegates CPU time among active tasks. To guarantee mutual exclusion without the processing overhead of busy-waiting loops, the binary semaphore interacts directly with the scheduling engine. When a running process requests a locked resource, the semaphore queues the thread and forces a transition into a BLOCKED state. Upon resource release, the system systematically dequeues the next pending task and elevates it back to a READY state. This deterministic synchronization model eliminates race conditions and provides a stable architectural foundation for subsequent operating system components.
